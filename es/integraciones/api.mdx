Recurrente cuenta con una API REST que te permite crear checkouts, gestionar suscripciones, consultar transacciones y mucho más desde tu propia plataforma o aplicación. Si los plugins de WooCommerce, Odoo o Bubble no se ajustan a tu caso de uso, la API te da control total para construir exactamente lo que necesitas.

## Cómo obtener tus llaves de API

Para autenticarte con la API necesitas dos llaves: una **llave pública** y una **llave secreta**. Para obtenerlas:

1. Ingresa a [app.recurrente.com](https://app.recurrente.com)
2. Ve a **Mi Cuenta → Desarrolladores y API**
3. En el espacio de **modo real (LIVE)**, haz clic en **Crear** para generar tus llaves. Si vas a integrar en pruebas primero, usa el espacio de **modo de prueba**.
4. Copia tu llave secreta (`sk_live_...` o `sk_test_...`) desde la ventana de confirmación. Solo se muestra completa una vez; después de recargar la página vuelve a mostrarse enmascarada.
5. Copia tu llave pública (`pk_live_...` o `pk_test_...`) desde el mismo espacio.

Desde esa misma sección también puedes configurar tus webhooks.

<Warning>
  Guarda tu llave secreta en un lugar seguro apenas la crees. Recurrente no vuelve a mostrarla completa; si la pierdes, tendrás que regenerarla.
</Warning>

## Tipos de llaves

| Llave | Prefijo | Uso |
|-------|---------|-----|
| Llave pública | `pk_live_` | Inicializar el SDK en el cliente (frontend) |
| Llave secreta | `sk_live_` | Llamadas desde tu servidor (backend) |

<Warning>
  **Nunca expongas tu llave secreta en el frontend ni en código público.** La llave secreta (`sk_live_`) solo debe usarse en tu servidor. Si alguien la obtiene, puede crear cobros en nombre de tu cuenta. Si sospechas que fue comprometida, regénérala inmediatamente desde **Mi Cuenta → Desarrolladores y API**.
</Warning>

## Autenticación

Todas las solicitudes a la API deben incluir tus llaves en los headers HTTP:

<Note>
  Incluye ambas llaves en cada solicitud. La llave secreta va en el header `Authorization` y la llave pública en `X-Public-Key`.

  ```http
  Authorization: Bearer sk_live_xxxxxxxxxxxx
  X-Public-Key: pk_live_xxxxxxxxxxxx
  ```
</Note>

## Ejemplo: crear un checkout

```bash
curl -X POST https://app.recurrente.com/api/checkouts \
  -H "Authorization: Bearer sk_live_xxxxxxxxxxxx" \
  -H "X-Public-Key: pk_live_xxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "items": [
      {
        "price_id": "price_xxxxxxxxxxxxxxxx"
      }
    ]
  }'
```

La API responderá con un objeto que incluye la `checkout_url` a la que debes redirigir a tu cliente para completar el pago.

Para explorar todos los endpoints disponibles, consulta la [documentación completa](https://docs.recurrente.com/).
