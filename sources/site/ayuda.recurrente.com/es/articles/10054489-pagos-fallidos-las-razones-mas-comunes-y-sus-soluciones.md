Los pagos en línea con tarjeta pueden fallar por diversos motivos, desde problemas con el banco hasta errores en la información de la tarjeta. En este artículo, exploraremos las causas más comunes de fallos en las transacciones y te daremos soluciones prácticas para resolver cada una.

# "Tu banco ha rechazado la transacción. Llama a tu banco y pide que autoricen esta transacción."

El banco es el intermediario principal entre tu cliente y Recurrente. Cada vez que tu cliente intenta hacer un pago con tarjeta, Recurrente solicita autorización al banco en fracciones de segundo. 
​

En algunos casos, el banco decide no autorizar la transacción, lo cual puede deberse a diversos motivos, por ejemplo:

- La transacción se sale de los patrones de consumo habituales de tu cliente, por lo que el banco la rechaza como medida de prevención.

- Algunos bancos extranjeros limitan transacciones realizadas hacia otros países.

- Pueden presentarse problemas técnicos en el sistema del banco.

La recomendación para este caso es:

1. **Explicarle al cliente que su banco ha rechazado la transacción.**

2. **Solicitarle que contacte a su banco para autorizar la transacción o pedirle que intente con una tarjeta distinta.**

# "No se pudo autenticar la transacción. Completa la autenticación o contacta a tu banco."

En Recurrente, estamos comprometidos a ofrecer una plataforma segura y confiable, por lo que implementamos la medida de seguridad 3D Secure (3DS) para reducir contracargos por fraude y validar cada transacción.

En algunos casos, este sistema antifraude requiere que el banco envíe un código de autenticación al cliente, ya sea por correo electrónico, mensaje de texto, aplicación de token, etc. Cuando el cliente no completa este proceso de autenticación, el pago falla y se despliega este error.

En ocasiones, el banco rechaza la transacción sin darle al cliente la oportunidad de realizar el proceso de autenticación.

La recomendación para este caso es:

1. **Indicarle al cliente que debe completar el proceso de verificación que el banco requiere.**

2. **Si el banco no le solicita autenticación pero aún aparece este error, sugerirle que contacte a su banco para autorizar la transacción o que intente con otra tarjeta.**

# "Tu intento de pago fue interrumpido. Por favor intenta de nuevo."

Cuando un cliente está en el proceso de autenticación 3D Secure y la ventana de autenticación se interrumpe (por ejemplo, si cierra la ventana del banco o la conexión se pierde), la página del checkout espera automáticamente a que el proceso se complete.

**Importante:** Si la autenticación no se completa en un plazo de 60 segundos, el sistema mostrará un mensaje indicando que el intento fue interrumpido y ofrecerá un enlace para intentar el pago nuevamente desde el inicio.

La recomendación para este caso es:

1. **Indicarle al cliente que haga clic en el enlace para iniciar un nuevo intento de pago.**

2. **Si el problema persiste, sugerirle que intente con una tarjeta diferente o que contacte a su banco para verificar que las transacciones en línea estén habilitadas.**

# "Tu sesión de pago ha expirado. Por favor ingresa nuevamente los datos de tu tarjeta."

Si tu cliente tarda más de 60 minutos en la página de pago sin completar la transacción, la sesión de pago expira por seguridad. Los datos de la tarjeta ya no son válidos y deben ingresarse nuevamente.

La recomendación para este caso es:

1. **Indicarle al cliente que vuelva a ingresar los datos de su tarjeta.**

2. **Si el problema persiste, sugerirle que abra una nueva ventana de checkout desde el enlace de pago.**

# "Esta tarjeta no es aceptada por Recurrente. Intenta con una tarjeta diferente."

Aunque estamos trabajando por incluir nuevos métodos de pago en nuestra plataforma, actualmente contamos con algunas limitaciones.

Al encontrarse con este error recomendamos:

1. **Revisar que la tarjeta sea Visa o Mastercard. No aceptamos American Express.**

2. **En el caso de pagos con cuotas, verificar que la tarjeta no pertenezca a los bancos BAC, Banco Azteca, Cuscatlán o Banco Antigua, ya que no aceptamos pagos con cuotas de esos bancos.**

# Algunos otros errores comunes

Existen otros errores que resultan en pagos fallidos y cuyos motivos son más claros, por lo que no suelen requerir mucha intervención con el cliente. Estos errores pueden originarse por error humano, restricciones impuestas por el banco, entre otros. Los errores son:

**"Código CVV inválido. Por favor intenta de nuevo."** Se ingresó incorrectamente el código de seguridad de 3 dígitos.

**"La fecha de tu tarjeta es incorrecta. Por favor inténtalo de nuevo."** Se ingresó incorrectamente la fecha de expiración de la tarjeta.

**"El número de tu tarjeta es inválido. Intenta de nuevo con una tarjeta diferente."** Se ingresó incorrectamente la enumeración de la tarjeta.

**"Información inválida. Revisa tu información e intenta de nuevo."** Puede desplegarse acompañado de los errores anteriores.

**"Tu tarjeta no está habilitada para transacciones en línea. Intenta de nuevo con una tarjeta diferente."** El banco no permite transacciones en línea para esta tarjeta.

**"Hubo un error con tu tarjeta. Intenta de nuevo con una tarjeta diferente."** Se recomienda contactar al banco o intentar con otra tarjeta.
